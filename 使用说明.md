# 进程隐藏工具 - 使用说明

## 📖 项目简介

这是一个带有图形用户界面的Windows进程隐藏工具，包含内核驱动程序和用户态客户端程序。通过该工具，可以将指定进程从系统进程列表中隐藏，使其不在任务管理器等工具中显示。

## 📁 项目结构

```
HideProcessesCheat-master/
├── 驱动部分 (Kernel Driver)
│   ├── Driver.c                    # 驱动主程序
│   ├── common.h                    # 公共头文件
│   ├── callbacks.h                 # 对象回调函数
│   ├── offset.h                    # 偏移量定义
│   ├── ObRegisterCallbacks.inf     # 驱动安装信息
│   └── ObRegisterCallbacks.vcxproj # 驱动项目配置
│
├── 客户端部分 (User-mode Client)
│   ├── Program.cs                  # 应用程序入口
│   ├── MainForm.cs                 # 主界面窗口
│   ├── ClientLib.cs                # 驱动通信库
│   ├── DriverService.cs            # 驱动管理服务
│   ├── hipro.csproj    # 客户端项目配置
│   └── app.manifest                # 应用清单（管理员权限）
│
└── hipro.sln          # Visual Studio解决方案
```

## 🔧 编译环境要求

### 驱动编译
- **Visual Studio 2019/2022**
- **Windows Driver Kit (WDK)** - 与您的VS版本匹配
- **Windows SDK**
- 目标平台：Windows 10 x64

### 客户端编译
- **.NET Framework 4.7.2** 或更高版本
- Visual Studio 2019/2022（包含.NET桌面开发工作负载）

## 🚀 编译步骤

### 方法一：使用Visual Studio（推荐）

1. **打开解决方案**
   ```
   双击 hipro.sln
   ```

2. **编译驱动**
   - 在解决方案资源管理器中右键点击 `hipro` 项目
   - 选择"生成"或按 `Ctrl+Shift+B`
   - 确保配置为 `Release | x64`
   - 编译成功后，驱动文件位于：`x64\Release\hipro.sys`

3. **编译客户端**
   - 右键点击 `hipro-Client` 项目
   - 选择"生成"
   - 可执行文件位于：`hipro-Client\bin\Release\hipro.exe`

### 方法二：使用MSBuild命令行

```powershell
# 进入项目目录
cd c:\Users\User\Downloads\HideProcessesCheat-master\HideProcessesCheat-master

# 编译驱动（需要WDK环境）
msbuild ObRegisterCallbacks.vcxproj /p:Configuration=Release /p:Platform=x64

# 编译客户端
msbuild hipro.csproj /p:Configuration=Release
```

## ⚙️ 使用方法

### 第一步：准备驱动程序

1. **禁用驱动程序签名强制**（测试模式）
   ```powershell
   # 以管理员身份运行PowerShell
   bcdedit /set testsigning on
   
   # 重启计算机使设置生效
   shutdown /r /t 0
   ```

2. **或者使用测试证书签名驱动**
   - 使用 `SignTool` 对驱动进行签名
   - 安装测试证书到受信任的根证书颁发机构

### 第二步：运行客户端程序

1. **以管理员身份运行**
   ```
   右键点击 hipro.exe
   选择"以管理员身份运行"
   ```
   
   > 程序会自动检测并要求管理员权限

2. **加载驱动程序**
   - 点击界面上的"加载驱动"按钮
   - 浏览并选择编译好的 `hipro.sys` 文件
   - 等待驱动加载成功（状态栏显示"驱动状态: 已加载 ✓"）

3. **查看进程列表**
   - 程序会自动加载当前所有运行的进程
   - 可以使用搜索框过滤进程
   - 点击"刷新进程列表"更新数据

4. **隐藏进程**
   - 在进程列表中选择一个进程
   - 点击"隐藏选中进程"按钮
   - 确认操作后，该进程将从任务管理器中消失

5. **卸载驱动**
   - 使用完毕后，点击"卸载驱动"按钮
   - 或重启计算机自动卸载

## 🎯 功能特性

### ✅ 已实现功能

1. **图形化界面**
   - 直观的Windows Forms界面
   - 实时显示驱动和连接状态
   - 进程列表可视化展示

2. **进程管理**
   - 自动枚举所有运行进程
   - 显示进程ID、名称和完整路径
   - 支持进程名称/PID搜索过滤

3. **驱动控制**
   - 一键加载/卸载驱动程序
   - 自动检测驱动状态
   - 安全的驱动通信机制

4. **进程隐藏**
   - 从系统进程链表中移除进程
   - 隐藏后进程对任务管理器不可见
   - 防止其他进程获取句柄

### ⚠️ 注意事项

1. **权限要求**
   - 必须以**管理员身份**运行客户端
   - 需要禁用驱动签名验证或使用签名驱动

2. **系统兼容性**
   - 仅支持 Windows 10/11 x64
   - 不同Windows版本的EPROCESS偏移可能不同

3. **安全性警告**
   - 此工具仅用于学习和研究目的
   - 隐藏关键系统进程可能导致系统不稳定
   - 某些安全软件可能将此行为标记为恶意

4. **恢复方法**
   - 重启计算机可恢复所有隐藏的进程
   - 卸载驱动不会自动恢复隐藏的进程链接

## 🔍 技术原理

### 驱动层（Kernel Mode）

1. **动态偏移计算**
   - 通过遍历内存查找`ActiveProcessLinks`字段
   - 适配不同Windows版本的EPROCESS结构

2. **进程链表操作**
   ```c
   // 从双向链表中移除节点
   pListEntry->Flink->Blink = pListEntry->Blink;
   pListEntry->Blink->Flink = pListEntry->Flink;
   pListEntry->Flink = 0;
   pListEntry->Blink = 0;
   ```

3. **设备通信**
   - 创建设备对象 `\\Device\\hipro`
   - 创建符号链接 `\\DosDevices\\hiproSL`
   - 处理IOCTL请求接收用户态命令

### 用户态（User Mode）

1. **驱动通信**
   - 使用`CreateFile`打开设备
   - 通过`DeviceIoControl`发送命令
   - 传递进程PID给驱动

2. **服务管理**
   - 使用SCM API创建和启动驱动服务
   - 监控驱动运行状态

3. **进程枚举**
   - 使用`.NET Process.GetProcesses()` API
   - 获取进程详细信息

## 🐛 故障排除

### 问题1：无法加载驱动

**症状**：点击"加载驱动"后报错

**解决方案**：
```powershell
# 1. 确认已禁用驱动签名强制
bcdedit | findstr testsigning

# 2. 如果不是on，执行：
bcdedit /set testsigning on
shutdown /r /t 0

# 3. 检查驱动文件是否完整
dir hipro.sys
```

### 问题2：连接驱动失败

**症状**：驱动已加载但显示"连接失败"

**解决方案**：
- 确认以管理员身份运行
- 检查设备名称是否正确
- 查看Windows事件日志中的驱动错误

### 问题3：编译驱动失败

**症状**：WDK相关错误

**解决方案**：
1. 确认已安装正确版本的WDK
2. 检查Visual Studio是否安装了"桌面开发"和"内核模式驱动开发"工作负载
3. 清理解决方案后重新编译

### 问题4：隐藏进程后系统不稳定

**解决方案**：
- 立即重启计算机
- 避免隐藏系统关键进程（如csrss.exe, svchost.exe等）
- 仅用于测试和学习目的
