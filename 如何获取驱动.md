## 📦 方法1：自己编译（推荐）

### 第一步：安装必要工具

#### 1.1 下载 Visual Studio 2022 Community（免费）

**下载地址**：
```
https://visualstudio.microsoft.com/zh-hans/downloads/
```

**安装步骤**：
1. 运行 `vs_Community.exe`
2. 在"工作负载"页面，勾选：
   - ✅ **使用C++的桌面开发**
   - ✅ **Windows驱动程序开发**（这会自动安装WDK）
3. 点击"安装"（大约需要5-10GB空间）

#### 1.2 或者单独安装WDK

如果Visual Studio已安装但没有WDK：

**WDK下载地址**：
```
https://docs.microsoft.com/windows-hardware/drivers/download-the-wdk
```

选择与你的WDK版本匹配的版本。

### 第二步：编译驱动

#### 2.1 打开项目
```
双击: hipro.sln
```

#### 2.2 配置编译选项
在Visual Studio顶部：
```
配置: Release
平台: x64
```

#### 2.3 编译
1. 在"解决方案资源管理器"（右侧）
2. 右键点击 **hipro**（不是HideProcessClient）
3. 选择 **"生成"**
4. 或按快捷键 `Ctrl+Shift+B`

#### 2.4 找到驱动文件
编译成功后，在以下位置：
```
HideProcessesCheat-master\x64\Release\hipro.sys
```

### 第三步：验证驱动
```powershell
# 检查文件是否存在
Test-Path ".\x64\Release\hipro.sys"

# 查看文件大小
Get-Item ".\x64\Release\hipro.sys" | Select-Object Name, Length
```

## 📥 方法2：下载预编译版本

### 从GitHub下载

1. 访问原项目仓库（如果有的话）
2. 在"Releases"页面下载最新版本
3. 解压后找到 `.sys` 文件

### ⚠️ 安全警告

下载预编译驱动时：
- ✅ 从可信来源下载（官方GitHub）
- ✅ 检查文件哈希值
- ❌ 不要从不明来源下载
- ❌ 使用前用杀毒软件扫描

## 🛠️ 快速检查清单

### 编译前检查

- [ ] 已安装 Visual Studio 2019/2022
- [ ] 已安装 Windows Driver Kit (WDK)
- [ ] 项目文件完整（Driver.c, callbacks.h等）
- [ ] 有足够的磁盘空间（至少5GB）

### 编译后检查

- [ ] 文件存在：`x64\Release\hipro.sys`
- [ ] 文件大小合理（通常几十KB）
- [ ] 可以用记事本打开（会看到乱码，说明是二进制文件）

## 📋 详细安装指南

### Visual Studio 2022 安装步骤

#### 1. 下载安装程序
```
https://visualstudio.microsoft.com/zh-hans/downloads/
下载: Visual Studio 2022 Community（免费）
```

#### 2. 运行安装程序
```
双击: vs_Community.exe
```

#### 3. 选择工作负载（重要！）

在"工作负载"标签页，勾选：

```
✅ Windows
  ├── ✅ 使用C++的桌面开发
  └── ✅ Windows驱动程序开发
      
✅ .NET
  └── ✅ .NET 桌面开发
```

#### 4. 单个组件（可选）

如果需要更精确控制，在"单个组件"中搜索并勾选：

```
✅ MSBuild
✅ C++ CMake tools for Windows
✅ Windows 10 SDK
✅ Windows 11 SDK
✅ Windows Driver Kit
```

#### 5. 安装位置
```
默认: C:\Program Files\Microsoft Visual Studio\2022\Community
需要空间: 约8-15GB
```

#### 6. 等待安装完成
```
时间: 30分钟 - 2小时（取决于网络速度）
```

## ❓ 常见问题

### Q1: 必须安装完整版Visual Studio吗？
**A**: 不必须，可以只安装"生成工具"（Build Tools），更轻量。

### Q2: WDK和Visual Studio版本必须匹配吗？
**A**: 强烈建议匹配，否则可能编译失败。

### Q3: 编译失败怎么办？
**A**: 
1. 检查WDK是否正确安装
2. 清理解决方案后重新编译
3. 查看错误日志

### Q4: 可以用旧版本的VS吗？
**A**: 可以，VS2019也可以，但建议用VS2022。
